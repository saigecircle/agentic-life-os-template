# Meta

The operating layer of the Life OS.

This folder explains how the system should work for humans and AI agents.

## What belongs here

- Personal operating context: `About Me.md`.
- Writing and communication preferences: `Voice.md`.
- Human and AI collaborators: `Team.md`.
- Tool routing and external systems: `Tools.md`.
- Agent behaviour, canonical workflows, adapters, and tool specs: `Agent OS/`.
- Templates or conventions that shape the whole vault.

## What doesn't belong here

- Active project work → `01 Projects/`.
- Person-specific relationship notes → `02 People/`.
- Ongoing responsibility content → `03 Areas/`.
- General reference material → `04 Resources/`.
- Secrets, credentials, tokens, or private keys → use a password manager.

## First files to customise

1. `About Me.md` — context an assistant should know before helping.
2. `Voice.md` — writing and communication preferences.
3. `Tools.md` — preferred tools, external systems, and routing rules.
4. `Agent OS/README.md` — how reusable agent workflows should be managed.

## Agent instructions

- Treat Meta as operating context.
- Read Meta files when the task involves preferences, tools, agents, reusable workflows, or system design.
- Keep reusable rules here instead of burying them in a single project or chat.
- Do not add tool-specific duplication when a canonical note can hold the rule once.
