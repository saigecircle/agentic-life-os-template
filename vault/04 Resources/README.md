# Resources

Reference material with no immediate action required.

Resources are useful things to consult later: frameworks, articles, research notes, examples, checklists, reusable explanations, tool notes, or saved context.

## What belongs

- Articles and reading notes.
- Research summaries.
- Frameworks and checklists.
- Reusable explanations.
- Examples that are not active project work.
- Tool notes that are informational rather than operational.

## What doesn't belong

- Active project work → `01 Projects/`.
- Ongoing responsibility context → `03 Areas/`.
- Relationship context → `02 People/`.
- Vault operating rules → `06 Meta/`.
- Raw captures that still need sorting → `00 Inbox/`.

## Recommended resource note shape

```markdown
---
type: resource
topic:
source:
---

# Resource Title

## Summary

What this is and why it is useful.

## Key takeaways

-

## Source / links

-

## Related notes

-
```

## Agent instructions

- Use Resources as reference, not as a task list.
- Link Resources to Projects or Areas when they support active work.
- Do not turn every interesting article into an Area or Project.
- Summarise long resources before copying large chunks into the vault.

## Naming

Use clear topic names. When date matters, use:

```text
YYYY-MM-DD - Topic.md
```
