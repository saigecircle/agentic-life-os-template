# Archive

Inactive items preserved for future reference.

The Archive keeps active work calm without pretending old context never existed.

## What belongs

- Completed projects.
- Cancelled projects worth preserving.
- Inactive areas.
- Superseded drafts.
- Old snapshots that may be useful later.
- Historical material that should not clutter active folders.

## What should stay active

- Current projects.
- Ongoing responsibilities.
- Reference material still used regularly.
- Person notes that remain useful relationship context.

## Structure

Projects archive by year:

```text
05 Archive/Projects YYYY/
```

Other categories can have their own folders:

```text
05 Archive/Areas Archive/
05 Archive/Misc Archive/
05 Archive/Superseded Drafts/
```

Use whatever categories make retrieval easier.

## Naming

When archiving with context, include the reason and date if useful:

```text
Original Name_reason_YYYY-MM-DD/
```

Examples:

```text
Website Launch_completed_2026-01-31/
Brand Drafts_superseded_2026-02-15/
```

## Agent instructions

- Ask before archiving important active material.
- Do not treat archived material as current without checking for a replacement or newer note.
- Preserve useful context when moving items into Archive.
- Do not delete just because something is old.

## What doesn't belong

- Active work.
- True junk that should be deleted.
- Secrets or credentials.
