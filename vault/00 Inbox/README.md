# Inbox

Capture-first parking zone. Ephemeral by design — nothing should live here forever.

> Friction-free capture during fast work, with regular light sweeps that move things to their real home.

## What belongs

- Quick captures: thoughts, links, half-formed notes.
- Unprocessed meeting notes or voice memo transcripts.
- Screenshots, downloads, and files that need sorting.
- Rough ideas that are not ready to become Projects, Areas, People, or Resources.
- Anything you would otherwise lose by not capturing immediately.

## What doesn't

- Anything that already has a clear home → file it directly.
- Project work → `01 Projects/`.
- Person notes → `02 People/`.
- Ongoing responsibility context → `03 Areas/`.
- Reference material → `04 Resources/`.
- Permanent operating rules → `06 Meta/`.
- Secrets, credentials, or private tokens → use a password manager, not the vault.

## Triage decision tree

When processing an inbox item, decide:

1. **Is this actionable with a finish line?** Move to or create a Project.
2. **Is this about a person or relationship?** Move to or update a People note.
3. **Is this about an ongoing responsibility?** Move to or update an Area.
4. **Is this reference only?** Move to Resources.
5. **Is this complete or inactive but worth keeping?** Move to Archive.
6. **Is this junk?** Ask before deleting unless you are the vault owner and are certain.

## Agent instructions

- Read the item before deciding where it belongs.
- File obvious items quickly.
- Ask when the destination would create a new long-lived folder, archive something important, delete something, or choose between equally plausible homes.
- Leave unclear incubating items in place and say why.
- Keep the report brief: moved, left, questions.

## Sweep behaviour

Inbox sweeps should be light and fast. This is a capture buffer, not an intelligence exercise.

- Do not over-analyse every capture.
- Apply simple rules consistently across a batch once approved.
- Do not turn every note into a project.
- If something keeps returning to the Inbox, define a better home or workflow.

## Naming

Free-form is fine for quick captures.

When dates matter, use:

```text
YYYY-MM-DD - Topic.md
```

Rename files on the way out of Inbox, not at the moment of capture.
