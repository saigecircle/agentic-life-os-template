# Folder Architecture

This template uses IP-PARAM:

- Inbox
- Projects
- People
- Areas
- Resources
- Archive
- Meta

## Inbox

Capture first, organise later.

## Projects

Time-bound work with a finish line. Each meaningful project should have a `README.md` and `Log.md`.

## People

Relationship context. Keep this respectful, useful, and private.

## Areas

Ongoing responsibilities with no finish line.

## Resources

Reference material with no immediate action required.

## Archive

Completed or inactive material.

## Meta

The operating layer. This is where the system explains itself to humans and agents.
