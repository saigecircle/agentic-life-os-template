# Privacy and Redaction

Never publish your real private vault.

Do not put these in a public repo:

- client notes;
- finance data;
- receipts, statements, invoices, or tax files;
- credentials, API keys, OAuth files, `.env` files, or tokens;
- session logs or exported chat histories;
- private URLs, hostnames, IP addresses, or infrastructure details;
- health, family, journal, or relationship notes;
- machine-specific app metadata.

Use fake examples only.

Before publishing, search for obvious sensitive terms like:

```text
api_key
secret
token
password
client
invoice
receipt
bank
private
```
